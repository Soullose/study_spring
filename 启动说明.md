# Spring Boot 多模块项目启动说明

## 项目结构概述

这是一个基于DDD架构的多模块Spring Boot项目，包含以下模块：

- [`start`](start/) - 启动模块（包含[`StartApplication.java`](start/src/main/java/com/wsf/StartApplication.java:23)主类）
- [`system`](system/) - 系统核心模块
- [`infrastructure`](infrastructure/) - 基础设施模块
- [`domain`](domain/) - 领域模块
- [`api`](api/) - API接口模块
- [`rest`](rest/) - REST控制器模块
- [`app`](app/) - 应用服务模块
- [`adapter`](adapter/) - 适配器模块

## 打包后启动方式

### 1. 打包整个项目

在项目根目录执行：

```bash
mvn clean package
```

### 2. 找到可执行JAR包

打包完成后，可执行JAR位于：
```
start/target/start-1.0-SNAPSHOT.jar
```

### 3. 启动服务

#### Windows系统：
```cmd
java -jar start/target/start-1.0-SNAPSHOT.jar
```

#### Linux/Mac系统：
```bash
java -jar start/target/start-1.0-SNAPSHOT.jar
```

### 4. 带参数启动

#### 指定配置文件：
```bash
java -jar start-1.0-SNAPSHOT.jar --spring.profiles.active=prod
```

#### 指定JVM参数：
```bash
java -Xmx512m -Xms256m -jar start-1.0-SNAPSHOT.jar
```

#### 指定端口：
```bash
java -jar start-1.0-SNAPSHOT.jar --server.port=8080
```

## 启动脚本示例

### Windows启动脚本 (`start.bat`)
```batch
@echo off
title Spring Boot Application
java -Xmx512m -Xms256m -jar start/target/start-1.0-SNAPSHOT.jar
pause
```

### Linux启动脚本 (`start.sh`)
```bash
#!/bin/bash
APP_NAME="start-1.0-SNAPSHOT.jar"
JAVA_OPTS="-Xmx512m -Xms256m"

echo "Starting Spring Boot Application..."
java $JAVA_OPTS -jar start/target/$APP_NAME
```

## 验证服务启动

启动成功后，控制台会显示类似信息：
```
----------------------------------------------------------
	应用程序"w2-server"正在运行中......
	接口文档访问 URL:
	本地: 		http://localhost:40001/doc.html
	外部: 		http://192.168.1.100:40001/doc.html
	配置文件: 	[default]
---------------------------------------------------------
```

## 注意事项

1. **Java版本要求**：项目使用Java 17，确保运行环境有JDK 17或以上版本
2. **配置文件**：配置文件位于[`start/src/main/resources/`](start/src/main/resources/)目录
3. **依赖模块**：所有依赖的模块会自动打包到可执行JAR中
4. **端口默认**：默认端口为40001，可在[`application.yml`](start/src/main/resources/application.yml:2)中修改

## 生产环境部署建议

1. 使用nohup或systemd后台运行
2. 配置JVM内存参数
3. 设置适当的日志级别和轮转
4. 配置健康检查端点监控
