# Spring Boot 多模块项目启动说明

## 项目结构概述

这是一个基于 DDD 架构的多模块 Spring Boot 项目，包含以下模块：

- [`start`](start/) - 启动模块（包含 [`StartApplication.java`](start/src/main/java/com/wsf/StartApplication.java) 主类）
- [`system`](system/) - 系统核心模块（历史遗留老式分层）
- [`infrastructure`](infrastructure/) - 基础设施模块
- [`domain`](domain/) - 领域模块
- [`api`](api/) - API 接口契约模块
- [`rest`](rest/) - REST 控制器模块
- [`app`](app/) - 应用服务模块
- [`adapter`](adapter/) - 适配器模块（预留）

## 环境要求

- **JDK 21**（以根 [`pom.xml`](pom.xml) 的 `maven.compiler.source=21` 为准，务必使用 JDK 21 或以上）
- Maven 3.6+
- MySQL（默认 `open` 库，见 [`application-datasource.yml`](start/src/main/resources/application-datasource.yml)）
- Redis（默认 127.0.0.1:6379，见 [`application-redis.yml`](start/src/main/resources/application-redis.yml)）

## 打包方式说明（重要）

本项目 `start` 模块**不是 Spring Boot 可执行 fat jar**，而是 **瘦 jar + `lib/` 依赖目录** 的形式：

- [`start/pom.xml`](start/pom.xml) 使用 `maven-jar-plugin` 生成普通 jar，并用 `maven-dependency-plugin` 把所有依赖复制到 `start/target/lib/` 目录。
- jar 的 `MANIFEST.MF` 中已写入 `Main-Class: com.wsf.StartApplication` 和 `Class-Path: lib/...`。
- spring-boot-maven-plugin 已被注释（见 `start/pom.xml` 末尾），所以**不会**生成 fat jar。

因此启动有两种方式，**推荐用脚本**：

| 方式 | 命令 | 说明 |
|------|------|------|
| 脚本（推荐） | `start.sh` / `start.bat` | 用 `-cp` 显式指定 jar + `lib/*`，最稳妥 |
| `java -jar` | `java -jar start/target/start-1.0-SNAPSHOT.jar` | 依赖 manifest 的 `Class-Path: lib/`，**必须在项目根目录执行**，且 `lib/` 与 jar 同级 |

> ⚠️ 注意：瘦 jar 不能脱离 `lib/` 目录单独拷贝运行，部署时 `start-1.0-SNAPSHOT.jar` 和 `lib/` 要一起带上。

## 1. 打包整个项目

在项目根目录执行：

```bash
mvn clean package
```

打包完成后产物：

```
start/target/start-1.0-SNAPSHOT.jar   # 瘦 jar（含启动类与配置）
start/target/lib/*.jar                # 全部第三方与模块依赖
```

## 2. 使用脚本启动（推荐）

### Linux / Mac

```bash
# 先赋予执行权限（首次）
chmod +x start.sh

./start.sh
```

### Windows

```cmd
start.bat
```

脚本会自动检查 jar 与 `lib/` 是否存在，并设置 JVM 内存参数后启动。脚本内默认 `-Xmx512m -Xms256m`，可直接编辑脚本调整。

## 3. 手动命令启动

### 方式一：`java -jar`（需在项目根目录）

```bash
# Linux/Mac
java -jar start/target/start-1.0-SNAPSHOT.jar

# Windows
java -jar start\target\start-1.0-SNAPSHOT.jar
```

### 方式二：`-cp` 指定 classpath（脚本采用的方式，任意目录可用）

```bash
# Linux/Mac
java -Xmx512m -Xms256m -cp "start/target/start-1.0-SNAPSHOT.jar:start/target/lib/*" com.wsf.StartApplication

# Windows（注意用分号分隔）
java -Xmx512m -Xms256m -cp "start\target\start-1.0-SNAPSHOT.jar;start\target\lib\*" com.wsf.StartApplication
```

## 4. 带参数启动

脚本和命令都支持在末尾追加 Spring Boot 参数（脚本会把参数透传）。

### 指定配置文件 / profile

```bash
./start.sh --spring.profiles.active=prod
# 或
java -jar start/target/start-1.0-SNAPSHOT.jar --spring.profiles.active=prod
```

### 指定端口

```bash
./start.sh --server.port=8080
```

### 指定 JVM 参数

脚本方式请直接编辑脚本里的 `JAVA_OPTS`；`java -jar` 方式如下：

```bash
java -Xmx1g -Xms512m -jar start/target/start-1.0-SNAPSHOT.jar
```

## 5. 验证服务启动

启动成功后，控制台会显示类似信息：

```
----------------------------------------------------------
	应用程序"w2-server"正在运行中......
	接口文档访问 URL:
	本地: 		http://localhost:40001/doc.html
	外部: 		http://192.168.1.100:40001/doc.html
	配置文件: 	[default]
----------------------------------------------------------
```

- 接口文档（Knife4j）：`http://localhost:40001/doc.html`
- 默认应用名：`w2-server`

## 注意事项

1. **Java 版本**：项目使用 **JDK 21**（以根 `pom.xml` 为准）。
2. **配置文件**：位于 [`start/src/main/resources/`](start/src/main/resources/)，按 profile 拆分（`application-jpa.yml`、`application-datasource.yml`、`application-redis.yml`、`application-openapi.yml` 等）。
3. **端口默认**：默认 `40001`，在 [`application.yml`](start/src/main/resources/application.yml) 中修改。
4. **依赖目录**：瘦 jar 依赖 `lib/`，打包、拷贝、部署时二者缺一不可。
5. **数据库/Redis**：启动前请确保 MySQL `open` 库与本地 Redis 已就绪。

## 生产环境部署建议

1. 用 `nohup` 或 `systemd` 后台运行，例如：
   ```bash
   nohup ./start.sh > app.log 2>&1 &
   ```
   或将 `-cp` 启动命令写入 systemd unit。
2. 根据服务器内存调整 `JAVA_OPTS`（`-Xmx` / `-Xms`）。
3. 配置合适的日志级别与轮转（见 [`logback-spring.xml`](start/src/main/resources/logback-spring.xml)）。
4. 配置健康检查端点监控（项目已引入 spring-boot-starter-actuator）。
5. 部署时把 `start-1.0-SNAPSHOT.jar` 与整个 `lib/` 目录一起发布。
