你是一位后端开发专家，需要完成一个基于 SpringBoot 3.5.0 的项目，具体要求如下：

1. **技术栈要求**：
   - 使用 Spring Security 实现基于角色的访问控制（RBAC），包含至少 3 种用户角色（如管理员、普通用户、访客）。
   - 使用 JPA 进行数据持久化，定义至少 5 个实体类，并配置双向关联关系。
   - 使用 QueryDSL 实现动态查询，支持至少 3 种复杂查询场景（如多条件过滤、分页、排序）。
   - 使用 Redisson 实现分布式锁和缓存，缓存至少 2 类高频访问数据，并设置合理的过期时间（如 10 分钟）。
   - 基于 JDK 21 开发，充分利用其新特性（如虚拟线程、模式匹配等）。

2. **网络通信**：
   - 使用 Netty 4.x 实现一个高性能 TCP 服务器，支持至少 1000 个并发连接，并处理自定义协议的数据解析。

3. **工具库应用**：
   - 使用 Guava 实现本地缓存，缓存容量不超过 1000 个条目，并设置过期策略。
   - 使用 Apache Commons 工具类处理字符串、集合等常见操作。
   - 使用 Bouncy Castle 实现 RSA , AES , SM2 , SM4 , RSA+AES混合 加密解密功能，确保数据传输安全。

4. **并发与线程池**：
   - 使用 JUC（Java 并发工具包）实现线程安全的计数器，并设计一个自定义线程池，核心线程数 5，最大线程数 20，队列容量 100。
   - 根据业务需求配置和优化线程池，包括但不限于核心线程数、最大线程数、队列类型及拒绝策略。

5. **编程风格**：
   - 使用 Java 流式编程（Stream API）处理集合数据，至少实现 3 个流式操作示例（如过滤、映射、归约）。

6. **架构与规范**：
   - 严格遵循 DDD（领域驱动设计）原则，划分领域层、应用层、基础设施层，确保每层职责清晰。
   - 代码注释覆盖率达到 90% 以上，每个公共方法需包含详细的注释说明。

7. **测试要求**：
   - 编写单元测试和集成测试，测试覆盖率不低于 80%，使用 JUnit 5 和 Mockito 进行测试。

请根据以上要求完成项目开发，并确保代码结构清晰、性能优化、符合规范。
